Architecture Documentation

Understanding the mechanics of BitHide.

1. The Encoding Engine

When you provide a carrier file (JPG, PNG) and a secret payload, BitHide first encrypts the text using AES-256 Symmetric Encryption. The encrypted binary cipher is then deeply embedded into the carrier using LSB (Least Significant Bit) manipulation. To the human eye and standard file metadata analyzers, the stego artifact appears completely unaltered.

2. The Extraction Process

By providing the correct stego file, our engine targets the LSB spectrum and fragments the manipulated bits back into a continuous data stream. It then validates the cryptographic signature using the AES passphrase provided. Once authenticated, the data is decrypted and presented natively in your browser.
